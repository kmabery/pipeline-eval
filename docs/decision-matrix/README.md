# Decision matrix — authoring guide

Source of truth: [`criteria.yaml`](criteria.yaml). All rendered tables (in [`../phase-plan.md`](../phase-plan.md), [`../evaluation-report.md`](../evaluation-report.md), and archived snapshots under [`../evaluation-reports/`](../evaluation-reports/)) live between `<!-- matrix:begin -->` / `<!-- matrix:end -->` markers and are regenerated by `node scripts/render-decision-matrix.mjs`. **Never hand-edit between the markers.**

## Criteria

| id | weight | group | why it matters |
|----|--------|-------|----------------|
| `approval` | 40 | core | Required in workflows — prove a deploy can be gated and traced to an approver. |
| `package` | 40 | core | npm and NuGet are required artifacts — both vendors must host or cache them. |
| `webhook` | 20 | agents | Pipeline monitoring — agents need a stable webhook delivery surface to react to runs. |

Weights must sum to **100**. Validation: `node scripts/validate-decision-matrix.mjs`.

## Vendor cell shape

Per criterion, per vendor:

```yaml
vendors:
  <vendor-key>:
    rating: pass | caveat | fail | tbd
    label: short summary (one line)
    citations:
      - https://docs.example.com/feature
    notes: longer evidence narrative; reference run URL / log excerpt / IaC artifact
    updatedInPhase: <phase-number>   # 1 for gha, 2 for ado; null for tbd
```

A non-`tbd` rating **must** include at least one citation URL.

## Write-lanes

| Phase | SDET edits | Forbidden |
|-------|-----------|-----------|
| 1 | `vendors.gha.*` for every criterion | `vendors.ado.*`; any top-level field |
| 2 | `vendors.ado.*` for every criterion | `vendors.gha.*`; any top-level field |
| 3 | Read-only — Analyst computes weighted signals | Any vendor cell |

Analyst owns criterion CRUD (`id`, `name`, `group`, `weight`, `whyItMatters`, `evidenceToCollect`, `commonNotes`) and rendering, in any phase.

See [`../../.cursor/rules/decision-matrix-authoring.mdc`](../../.cursor/rules/decision-matrix-authoring.mdc).

## 5-step authoring loop

After **any** edit to `criteria.yaml`:

```bash
node scripts/validate-decision-matrix.mjs
node scripts/render-decision-matrix.mjs --target phase-plan
node scripts/render-decision-matrix.mjs --target index
node scripts/render-decision-matrix.mjs --target snapshot --path docs/evaluation-reports/<file> --phase <n>
node scripts/validate-phase-evidence.mjs
```

Phase 3 adds `--final` to the snapshot render.
