## How to add a new snapshot

1. Run the appropriate phase launcher under [`.cursor/plans/phases/`](../.cursor/plans/phases/).
2. After SDET + DevOps work completes, the Analyst writes a new snapshot under [`evaluation-reports/`](evaluation-reports/) with the filename pattern **`evaluation-report-MM-dd-yyyy-HHmmss-phase-<n>[-<vendor>-<runId>].md`** (UTC time stamp). Example: `evaluation-report-04-27-2026-093500-phase-1-gha-12345.md`.
3. Each snapshot contains **only** these sections in order: `## Executive summary` (SHORT — problems and deltas vs. the previous run), `## Link bundle`, `## Rendered decision matrix` (between `<!-- matrix:begin -->
| Criterion | Weight | Why it matters | GitHub Actions | Azure DevOps |
|---|---|---|---|---|
| `approval` | 40 | Required in workflows | tbd | tbd |
| `package` | 40 | npm and NuGet are required artifacts | tbd | tbd |
| `webhook` | 20 | monitoring success of pipelines | tbd | tbd |
<!-- matrix:end -->` markers, populated by `node scripts/render-decision-matrix.mjs --target snapshot --path <file> --phase <n>`).
4. Update the **Latest archived report** table above and re-render this index: `node scripts/render-decision-matrix.mjs --target index`.
